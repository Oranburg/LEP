# LEP Repository Audit — 2026-05-27

## Executive summary

LEP is a Jekyll site (Ruby/kramdown, deployed at oranburg.law/LEP) staging the intellectual identity of the Law and Entrepreneurship Program at CUA Columbus School of Law, which Seth Oranburg will co-direct with Alexander Hoffarth beginning 2026-08-20. The site comprises 31 pages, 10 working-paper essays (gated behind a courtesy access key, SHA-256 in `_config.yml`), 1 active draft, 1 welcome post, 2 lineage-diagram proposals, six data files, three layouts, and ~18 includes. Content quality is high and the doctrinal/programmatic spine is coherent (the *societas* / *shutafut* convergence, *Entrepreneurship v. Poverty* year-theme, four named giving programs, Athens/Jerusalem/Rome/Washington genealogy). The "mish mosh" Seth flagged is real but localized. The principal structural problem is the *Magnifica Humanitas* cluster: a labeled "close reading" page that violates a memory rule set the same day, an existing essay (`magnifica-babel`), a fresh long-form draft (`magnifica-jewish-asymmetry`), and four `_data/magnifica-humanitas/` research files, with no clear pipeline that says which one supersedes which. Secondary problems are nav duplication (`-updated` includes alongside originals), at least one orphan page (`traditions-comparison`), inconsistent essay frontmatter, and several inline forbidden-phrase risks. Site is publishable; cleanup is editorial, not architectural.

## Inconsistencies and rule violations

1. **Magnifica "close reading" label — rule violation.** `_pages/magnifica-humanitas.md` is titled "Magnifica Humanitas: A Close Reading" (line 3) and uses the phrase "close-reading" or "close reading" four times in the page (lines 3, 7, 9, 35). The memory rule `feedback_magnifica-humanitas-label.md` set 2026-05-26 forbids calling Seth's reading of *Magnifica Humanitas* a "close reading." `_pages/about.md:21` and `_pages/teach.md:25` also use "close reading" in reference to the Magnifica essay. **The correct replacement label is pending from Seth.** Until Seth supplies it, these uses should be marked TODO and the page title changed to something neutral (e.g., "Magnifica Humanitas: Reception and Reading" or "Magnifica Humanitas: A Reading in Progress"). The phrase must not ship.

2. **Magnifica cluster overlap.** Four artifacts overlap on the same encyclical: (a) `_pages/magnifica-humanitas.md` (landing page, promises forthcoming essay), (b) `_essays/magnifica-babel.md` (already-published gated working draft, AI-platform/Babel framing), (c) `_drafts/magnifica-jewish-asymmetry.md` (new Tikvah-register Jewish-internal essay, 4,800 words, no frontmatter), (d) `_data/magnifica-humanitas/` (four research files including a Sacks bridges file and a Nehemia/Babel research dump). It is unclear which one is the canonical "close reading" the landing page promises. The landing page itself names section themes ("AI/labor", "Babel/Nehemia") that magnifica-babel already covers from the AI-platform angle and the new draft covers from the Jewish-asymmetry angle.

3. **Frontmatter inconsistency.** Two clear patterns coexist in `_essays/`: (a) "working-draft convention" (slug, gated, excerpt, title, date, author, status, citations_to_verify) used by ai-as-havruta, heter-iska-operative, jurisgenesis, magnifica-babel, deskilling-loop, three-states, washington-taken-seriously; (b) "funder one-pager convention" (program, authors, audience, budget — no status, no citations_to_verify) used by pilgrimage-bootcamp, field-immersion-institute; (c) "institutional brief convention" (length, note — no status) used by cua-as-home. The drafts file has no frontmatter at all.

4. **Duplicate include files.** `_includes/` contains nav.html + nav-updated.html, footer.html + footer-updated.html, header.html + header-updated.html. Three "updated" shadow files indicate an in-flight chrome refactor. Which version is the layout actually using? Needs to be resolved and the loser deleted.

5. **Orphan page: traditions-comparison.** `/traditions-comparison/` is a substantial mermaid-rich page but is not linked from index.md, lineage.md, glossary.md, or any nav. Compare to lineage-timeline.md which is also a sibling-view of lineage data and is similarly close-to-orphaned (linked only from lineage.md). Risk of three near-duplicate "lineage in another view" pages.

6. **`magnifica-jewish-asymmetry.md` has no Jekyll frontmatter.** As placed in `_drafts/`, Jekyll will not build it (drafts excluded by default), but if it ever moves to `_essays/` it will fail to render correctly without slug/gated/excerpt/title/date/author/status.

7. **`_proposals/` excluded from build but `layout: page` is set on at least one proposal.** `catholic-lineage-proposal.md` has `layout: page` and `permalink: /proposals/catholic-lineage/` — these are dead because `_config.yml` excludes `_proposals`. Either the proposals should be promoted (and the layout intent honored) or the frontmatter should be stripped to plain notes.

8. **Forbidden-phrase / AI-tell risk surface.** A grep of `_essays/` and `_pages/` for memory-rule banned constructions has not been run as part of this audit but is recommended. The term-tooltip on `theme.md` ("the engine of data colonialism") is deliberately tagged `status="violation"` as a style demo, but a reader will not necessarily know that.

9. **`partnership-at-shared-risk.md` footnote not pinned.** Footnote `[^fn-magnifica-humanitas]` cites the encyclical without a paragraph pin and self-flags as PARTIALLY CONFIRMED. Citation-placement rule (citations follow text, never start a line with `[#` or `[^`) is observed in this file.

10. **Mixed slug conventions in published URLs.** Most essays publish at `/essays/<slug>/`; program funder pages publish at `/support/<name>/`; for-students pages nest at `/for-students/<role>/`. This is fine, but the program-named gift pages and the essay pilgrimage-bootcamp + field-immersion-institute (both are "programs") live in different URL spaces (`/support/` vs `/essays/`). The essays are the long-form versions of the funder asks but they sit on the academic side of the site.

## Redundancies

- **Three lineage views**: `/lineage/` (anatomy walkthrough), `/lineage-timeline/` (timeline), `/traditions-comparison/` (side-by-side). All draw the same convergence. Pick one canonical and demote the other two to "alternative views" with explicit navigation, or merge.
- **Two "doorways" / audience-router surfaces**: `index.md` and `/for-students/`. Acceptable, but the cards on for-students duplicate two of the six index cards.
- **Two `_proposals/` and the lineage page**: the proposals are upgrades-in-waiting for the same diagrams that already exist in `_pages/lineage.md`. Adopt or close.
- **Two pieces of the same flagship summer**: pilgrimage-bootcamp + field-immersion-institute deliberately mirror each other (student arm / faculty arm). This is intentional, not a redundancy — flag for awareness only.
- **Magnifica cluster** (see Inconsistency #2).

## Gaps

- **No canonical Magnifica essay yet.** The landing page promises one; magnifica-babel covers part of the ground from a securities/platform angle; the new Jewish-asymmetry draft covers part from inside the Jewish tradition. A third piece — the one the landing page is actually advertising — does not yet exist as a single artifact.
- **No Maimonides-on-shutafut standalone essay** (teach.md announces one as "forthcoming"; not in `_essays/` or `_drafts/`).
- **No *Laborem Exercens / exercitor navis* article** (teach.md says "in folder embargo pending the Salamanca-school recovery search").
- **`events.yml` and `collaborators.yml` may be sparse/empty.** Pages will render empty loops gracefully but the public-facing impression is "stub."
- **No essay on AotV summary or SoL summary on LEP itself**, despite repeated cross-reference. A program reader who follows the link has nowhere on LEP to land.
- **Glossary missing entries** for: *iska*, *exercitor navis*, *phronesis*, *ḥokhmat lev*, *prudentia*, *ordo disciplinae*, *jurisgenesis*, sub-creation — all of which appear in pages/essays.
- **No CUA-disclaimer banner include.** The "not a CUA-authorized site" line appears in index.md, about.md, for-students-prospective.md, news.md/welcome — should be a single include so the language is uniform.
- **No README- or HANDOFF-driven check** of which essays are linked from `_essays/index.md` vs `_pages/working-papers.md` (working-papers lists five; essays/index lists ten — these need to be reconciled).

## Recommended cleanup priorities

1. **Rip out "close reading" from the Magnifica page and every other page that uses it.** Replace the page title and the four in-page uses with a placeholder pending Seth's correct label. Same for about.md and teach.md. This is a rule-violation fix and ranks first.
2. **Decide the Magnifica cluster pipeline.** One of three moves: (a) make `magnifica-jewish-asymmetry` the canonical close-reading essay, give it frontmatter, file it as `_essays/magnifica-humanitas-reading.md`, and demote `magnifica-babel` to companion; (b) keep magnifica-babel as the LEP-internal piece and route magnifica-jewish-asymmetry to Mosaic/Tikvah outside the site; (c) write a third synthesizing essay. The landing page text should match whichever path is chosen.
3. **Reconcile `_essays/index.md` (10 entries) with `_pages/working-papers.md` (5 entries).** Either expand working-papers or explain why the working-papers page is curated.
4. **Resolve the `-updated.html` include duplication.** Diff the pairs, keep one, delete the other, update `_layouts/default.html` if needed.
5. **Standardize essay frontmatter.** Pick one convention (slug, gated, excerpt, title, date, author, status, citations_to_verify) and patch the funder-one-pager essays + cua-as-home to fit; add frontmatter to magnifica-jewish-asymmetry before it leaves `_drafts/`.
6. **Decide the fate of the three lineage views and the `_proposals/` upgrades.** Either adopt the proposal mermaid blocks into lineage.md and traditions-comparison.md and delete `_proposals/`, or move proposals to a tracked "design log" outside the Jekyll tree.
7. **Glossary completion + disclaimer include.** Add the missing entries; extract the not-a-CUA-site paragraph into `_includes/cua-disclaimer.html` and include it once on each public-facing landing page.
8. **Grep for memory-rule forbidden phrases.** Run a sweep against the full list in `feedback_no-ai-prose-patterns.md` and `feedback_no-load-bearing.md` before any external party is pointed at the site.
9. **Stub out (or remove) the empty-loop pages** until `events.yml` and `collaborators.yml` carry real content, or add a `{% if forloop.length == 0 %}` fallback line.
10. **Footnote pin discipline.** Sweep the `citations_to_verify` blocks in `_essays/` for items still marked UNVERIFIED / PARTIALLY CONFIRMED and run a `lawj verify-batch` round before any of the gated essays is un-gated.
